# jamfpro_user (Data Source)

The `jamfpro_user` data source allows you to look up Jamf Pro users by ID, username, or email address. This is useful for retrieving user IDs for use in static user groups or other resources that reference users.

## Example Usage

```terraform
# Lookup user by username
data "jamfpro_user" "user_by_name" {
  name = "john.doe"
}

# Lookup user by email
data "jamfpro_user" "user_by_email" {
  email = "john.doe@example.com"
}

# Lookup user by ID
data "jamfpro_user" "user_by_id" {
  id = "123"
}

# Use in a static user group
resource "jamfpro_user_group" "engineering_team" {
  name     = "Engineering Team"
  is_smart = false

  assigned_user_ids = [
    tonumber(data.jamfpro_user.user_by_name.id),
  ]
}

# Output user details
output "user_info" {
  value = {
    id            = data.jamfpro_user.user_by_name.id
    name          = data.jamfpro_user.user_by_name.name
    full_name     = data.jamfpro_user.user_by_name.full_name
    email_address = data.jamfpro_user.user_by_name.email_address
    position      = data.jamfpro_user.user_by_name.position
  }
}

# Verify that different lookup methods return the same user
output "matching_ids" {
  value = (
    data.jamfpro_user.user_by_name.id == data.jamfpro_user.user_by_email.id &&
    data.jamfpro_user.user_by_email.id == data.jamfpro_user.user_by_id.id
  )
}
```

## Lookup Methods

You must specify exactly **one** of the following lookup fields:

- `id` - The unique identifier of the user
- `name` - The username/login name of the user
- `email` - The email address of the user

Specifying multiple lookup fields or omitting all of them will result in an error.

## Note on User IDs

The `id` field is returned as a string to comply with Terraform provider conventions. When using the user ID in resources that expect integer values (such as `jamfpro_user_group.assigned_user_ids`), you must convert it using the `tonumber()` function:

```terraform
assigned_user_ids = [
  tonumber(data.jamfpro_user.example.id),
]
```
